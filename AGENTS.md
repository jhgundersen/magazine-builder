# Project Notes

## What this is

A local web app for building AI-generated print-style magazines, newspapers, comics, zines, and similar publications.

The intended workflow is:
1. The user describes a publication style and optional reference image
2. `defapi text` turns that into compact reusable `magazineStyle` JSON, including a `colorPalette`
3. Articles are imported from RSS/Atom, generated, or entered manually (including poster pages)
4. Article bodies are rewritten into the chosen publication voice (posters are not rewritten)
5. The app creates an issue plan, issue metadata, creative kit, brand asset board, page prompts, rendered page images, and finally a PDF

The app intentionally supports strong formats such as comics, satire, tabloids, puzzle pages, listings, letters, classifieds, and other non-standard magazine structures. Do not flatten those into generic respectable magazine copy.

## Source files

### `main.go`
Server entry point. Defines `config`, `server`, context key types, `main()`, `routes()`, and `parseFlags()`. All HTTP routes are registered here. No business logic.

### `style.go`
Core data types for style and identity:
- `magazineStyle` — the central style contract passed everywhere. Fields: language, tone, core visual description, per-page-kind visual notes (cover, feature, short, advert, filler, back), typography, color description, print treatment, avoid list, and a structured `colorPalette`.
- `colorPalette` — five CSS hex strings: primary, secondary, accent, background, text. Generated by `enhanceStyle` during style enhancement and injected as structured JSON into every image prompt. Do not use a palette strip image; hex codes in JSON are precise.
- `creativeKit` — reusable issue-wide module pools: departments, adverts, sidebars, back-page ideas. Not for captions.
- `issueContext` — issue number, year, date string, label. Chosen once per plan, reused everywhere.
- `styleBrief`, `brandAsset` — supporting types.
- Style parsing, normalization, fallbacks (`fallbackStyle`, `normalizeStyle`, `decodeStyle`). Comic/Norwegian style detection lives in `fallbackStyle`.
- `issueContextLine`, `newIssueContext`, `normalizeIssueContext`.
- `styleLine`, `styleLineSpecific` — build per-kind style strings for prompts.
- `extractJSONObject` — finds the outermost JSON object in LLM output.

### `prompts.go`
Page planning and image prompt construction:
- `pagePlan` — describes one page: number, kind, title, prompt, images, article reference.
- `pageFurniture` — header (section slug) + footer (pub name / short label).
- `modulePlanner` — cycles through creative kit pools without repeating within a kind.
- `planMagazine` — assembles the full page plan from a build request. Handles cover (page 1), back page (last page), advert slots, poster pages, multi-page article spreads, and filler pages.
- `coverPrompt`, `articlePrompt`, `genericPrompt`, `posterPrompt` — build image prompts as compact JSON strings. All include `"palette": style.Palette` in the style block.
- `imageStyleBrief` — assembles a per-page visual brief from style fields.
- `pageFormatInstruction` — canonical format/aspect-ratio instruction string.
- `pageSide`, `pageNumberSide` — left/right page helpers.
- `isAdvertPage`, `normalizePageCount` — page planning helpers.
- `pageListForCover`, `cleanCoverLines`, `fallbackCoverPlan` — cover line assembly.
- `pageBodyForFurniture`, `fallbackPageFurniture` — furniture fallback logic.

### `ai.go`
All `defapi` calls and AI-driven generation:
- `enhanceStyle` — asks `defapi text` to turn a raw style description into full `magazineStyle` JSON. Requires `palette` key with five hex color values.
- `generateIssueContext` — picks issue number, year, date once per build.
- `generateCreativeKit` — generates the reusable module pools.
- `generateBrandAssets` — generates the brand asset reference image (masthead, wordmark, issue number mark `"No. N"`, divider). Issue seal shows only the issue number — no date or year. Returns the public URL used as a reference image in page renders.
- `generatePageFurniture` — calls `defapi text` to produce header + footer. Header is always a section/department slug (2–4 words, e.g. "Features", "Interview"). Footer is the publication name or short section label. Issue metadata (number/year/date) must not appear in the header or footer.
- `pagePromptWithFurniture` — appends furniture JSON into a page's image prompt. Cover and poster pages skip furniture entirely.
- `generateCoverPlan` / `refineCoverLines` — cover line planning.
- `rewriteManualArticleForStyle`, `rewriteFeatureForStyle`, `rewriteArticleForStyle`, `summarizePodcastForImport` — article rewrite variants.
- `generateArticles` — generates articles from scratch given a title and style.
- `runDefapiText`, `runDefapiImage`, `runDefapiImageWithRetry` — low-level `defapi` subprocess wrappers.
- `brandAssetRefsForPage` — selects the right brand asset image(s) for a given page kind.

## Page kinds

| Kind | Description |
|------|-------------|
| `cover` | Page 1. No furniture. AI picks dominant image + cover lines. |
| `article` | Standard content page. Has headline, body columns, pull quotes. |
| `feature` | Visually dominant article page. Larger hero image, stronger hierarchy. |
| `poster` | Full-page image page. Body is the user's image description; no text columns; no LLM rewrite; no furniture. |
| `advert` | Auto-inserted at fixed slots (mid-issue and near-back). Fictional brands only. |
| `filler` | Department/briefs pages at the end of the article sequence. |
| `back-page` | Last page. Subscription panel, teaser, classifieds, or closing visual. |

## Key design rules

- **Style JSON is the contract.** `magazineStyle` must include language, tone, all visual notes, `articleLength`, and `palette`. Do not hardcode length or color rules — let `defapi text` derive them.
- **Color palette is hex in JSON.** Do not use a palette strip image. The `colorPalette` struct (primary, secondary, accent, background, text) is generated during style enhancement and injected into every image prompt as structured JSON.
- **Issue identity is chosen once.** Issue number, year, date, and label are fixed at build time by `generateIssueContext`. They must not drift across pages.
- **Brand asset board has four elements:** masthead, wordmark, issue number mark (`"No. N"` only — no date or year), divider. At most one input image slot consumed per page render.
- **Page furniture = section slug + pub name.** Header is a 2–4 word section/department label. Footer is the publication name or a short label. Issue metadata must not appear in either. Cover and poster pages have no furniture.
- **Posters skip rewriting.** The user's image description goes directly to the image generator. Posters always occupy exactly one page.
- **Manual article rewrites are aggressive.** RSS rewrites preserve source facts; manual rewrites may add editorial texture when material is thin.
- **Creative kit is for reusable modules only.** Not for per-image captions. Image text is generated per page and carries no label prefixes.
- **Defapi image input slots are limited.** Reserve slots for article images. Brand assets should cost at most one slot.

## Dev server

Free port 8080 first, then start the app on `:8080`. Do not keep moving to new ports for stale processes.

```
# Find and stop the process using :8080
make run ADDR=:8080
```

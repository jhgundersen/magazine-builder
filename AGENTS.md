# Project Notes

- This is a local web app for building AI-generated print-style magazines, newspapers, comics, zines, and similar publications.
- The intended workflow is:
  - the user describes a publication style and optional reference image
  - `defapi text` turns that into compact reusable style JSON
  - articles are imported from RSS/Atom, generated, or entered manually
  - article bodies are rewritten into the chosen publication voice
  - the app creates an issue plan, issue metadata, creative kit, brand asset board, page prompts, rendered page images, and finally a PDF
- The app intentionally supports strong formats such as comics, satire, tabloids, puzzle pages, listings, letters, classifieds, and other non-standard magazine structures. Do not flatten those into generic respectable magazine copy.
- The style JSON is a core contract. It should include publication language, tone, visual/page guidance, and `articleLength`, which tells rewrite prompts how long and structurally dense article bodies should be for that publication. Avoid hardcoding format length rules when defapi text can infer them into style JSON.
- Manual or partial articles should be developed more aggressively into the magazine voice than RSS imports. RSS rewrites should preserve source facts and chronology; manual rewrites may add plausible connective tissue, framing, jokes, departments, or editorial texture when the user's material is thin.
- Creative kit JSON is for reusable issue-wide modules only: departments, adverts, sidebars, and back-page ideas. Do not add reusable captions to the creative kit. Image text should be generated per article/page and should not include label prefixes such as caption labels.
- The issue number/date/year must be chosen once per plan and reused everywhere. It can be generated by `defapi text` from the original style prompt and should not drift between pages.
- Brand assets are generated as one public image reference, currently an unlabeled asset board with masthead, header wordmark, issue seal, divider/rule, and folio marks. Render prompts should use at most one element from that board per page and must not reproduce the whole board or explanatory labels.
- Defapi image supports a limited number of input images. Preserve room for article images; brand assets should generally cost at most one input image per render.
- When restarting the local development server, free port 8080 first and then start the app on `:8080`. Do not keep moving to new ports for stale server processes.
- Useful restart pattern:
  - find the process using `:8080`
  - stop that process
  - run `make run ADDR=:8080`
